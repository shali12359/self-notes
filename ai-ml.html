Nerd-Nest

1. Machine Learning Fundamentals

Progress:

  • Artificial Intelligence (AI): The broad field of making machines perform tasks that usually require human intelligence (e.g., reasoning, problem-solving, decision-making).
  • Machine Learning (ML): A subset of AI where systems learn patterns from data instead of being explicitly programmed. Example: predicting house prices from past sales data.
  • Deep Learning (DL): A subset of ML that uses neural networks with many layers to learn complex patterns, often from large amounts of data (e.g., image recognition, speech-to-text).
  • Think of it as: AI is the big umbrella → ML is one approach under AI → DL is a powerful technique within ML.

  • Supervised Learning: Data has input + labeled output. The model learns to map input to output. Example: Email spam detection (email text → spam/ham).
  • Unsupervised Learning: Data has input only, no labels. The model finds hidden patterns or groupings. Example: Customer segmentation (grouping buyers by behavior).
  • Reinforcement Learning (RL): The model learns by trial and error, receiving rewards/penalties for actions. Example: A robot learning to walk by balancing rewards for stable steps.

  • Regression: Predicting a continuous value. Example: Predicting house prices ($200k, $300k).
  • Classification: Predicting a category/class. Example: Predicting whether an image is a cat, dog, or bird.
  • Clustering: Grouping similar data points without labels. Example: Grouping news articles into topics automatically.

  • Overfitting: The model learns training data too well, including noise, and performs poorly on unseen data. Example: Memorizing exam questions instead of learning concepts.
  • Underfitting: The model is too simple and fails to capture important patterns in the data. Example: Trying to fit a straight line to data that clearly curves.
  • Goal: Find the right balance so the model generalizes well.

  • Bias: Error from overly simplifying the model (underfitting).
  • Variance: Error from the model being too sensitive to training data (overfitting).
  • Tradeoff: Increasing complexity reduces bias but increases variance, and vice versa.
  • Goal: Achieve the sweet spot where both bias and variance are balanced.

  • Train set: Used to teach the model.
  • Validation set: Used to tune hyperparameters and check performance during training.
  • Test set: Used to evaluate final performance on unseen data.
  • Cross-validation: Instead of one validation split, data is split into k folds. Each fold gets a chance to be validation, and results are averaged → gives a more reliable performance estimate.
  • Helps: Avoid bias from a single lucky/unlucky split of data.

2. Machine Learning Algorithms

Progress:

  • Data has inputs (features) and outputs (labels).
  • The algorithm learns a mapping from inputs → outputs.
  • Example: Predicting house prices from size, location, and features.
  • Algorithms:
    1. Linear & Logistic Regression
      • Linear Regression: Predicts continuous values (e.g., house price). Uses a straight-line equation.
      • Logistic Regression: Predicts probabilities for categorical outcomes (e.g., spam/not spam). Uses a sigmoid function for classification.
    2. Decision Trees & Random Forests
      • Decision Trees: Models decisions as a tree of rules (if-else splits). Easy to interpret but can overfit.
      • Random Forests: An ensemble of many decision trees. Results are averaged/voted, reducing overfitting and improving accuracy.
    3. Support Vector Machines (SVM)
      • Finds the best separating boundary (hyperplane) between classes.
      • Works well in high-dimensional spaces.
      • Can use kernels to handle non-linear decision boundaries.
    4. k-Nearest Neighbors (kNN)
      • Classifies a new point by looking at the k closest neighbors in training data.
      • The majority class among neighbors is assigned.
      • Easy to understand but slow for large datasets.

  • Data has no labels.
  • The algorithm finds hidden patterns or groups.
  • Example: Customer segmentation by purchasing behavior.
  • Algorithms:
    1. k-Means & Hierarchical Clustering
      • k-Means: Groups data into k clusters by minimizing distance to cluster centers.
      • Hierarchical Clustering: Builds a tree of clusters (agglomerative or divisive). Good for visualizing relationships.
    2. PCA (Principal Component Analysis)
      • A dimensionality reduction technique.
      • Converts correlated features into fewer principal components while keeping most variance.
      • Useful for visualization, noise reduction, and speeding up models.
    3. Anomaly Detection
      • Identifies rare or unusual patterns in data.
      • Commonly used in fraud detection, network intrusion detection, and faulty sensor readings.

  • An agent learns by interacting with an environment.
  • Receives rewards/penalties for actions and learns a policy to maximize rewards.
  • Example: Training a robot to walk or a game AI to play chess.
  • Algorithms:
    1. Q-Learning & MDP
      • MDP: A framework for decision-making where outcomes depend on both chance and actions.
      • Q-Learning: Learns the value (Q-value) of taking an action in a state, guiding toward the best long-term rewards.
    2. Ensemble Methods
      • Combine multiple models to improve accuracy.
      • Reduces variance and bias compared to single models.
      • Bagging: Trains models in parallel on random subsets (e.g., Random Forest).
      • Boosting: Trains models sequentially, focusing on errors of previous ones (e.g., AdaBoost, XGBoost).

3. Neural Networks & Deep Learning

Progress:

  • A neural network is inspired by how the brain works.
  • It consists of layers of neurons connected by weights.
  • Each neuron: input → weighted sum + bias → activation function → output.
  • Networks learn by adjusting weights during training to capture complex patterns.

  • Perceptron: Simplest neuron (weighted sum + bias + activation).
  • Activation Functions:
    • ReLU: Fast, outputs 0 if input < 0, else input.
    • Sigmoid: Maps values to [0,1], used in binary classification.
    • Tanh: Maps values to [-1,1], centered at 0.

  • Feedforward: Data flows input → hidden layers → output.
  • Backpropagation: Error (predicted – actual) flows backward, updating weights with gradient descent.
  • Training = repeated feedforward + backprop until good performance.

  • Specialized for image data.
  • Use convolution filters to detect edges, textures, shapes.
  • Structure: Convolution → Pooling → Fully Connected → Output.
  • Applications: Image classification, object detection, face recognition.

  • RNN: For sequential data, remembers past context.
  • Problem: Vanishing gradients → forgets long dependencies.
  • LSTM: Adds gates (input, forget, output) → solves long-term memory issue.
  • Applications: Text, speech, stock prediction, time-series analysis.

  • Use Attention Mechanism → model focuses on important words.
  • Unlike RNNs, transformers process all tokens in parallel.
  • Example: "The dog chased the cat because it was hungry" → attention helps link "it" to "dog".
  • Applications: NLP (BERT, GPT, T5), translation, summarization.

  • Dropout: Randomly disable neurons during training.
  • Regularization: Penalize large weights (L1, L2).
  • Other methods: Early stopping, data augmentation.

4. Data Preprocessing

Progress:

  • Real-world datasets often have missing values (e.g., surveys with unanswered questions).
  • Approaches:
    • Remove rows/columns with too many missing values.
    • Imputation:
      • Mean/median (numeric).
      • Mode (categorical).
      • Advanced: KNN or regression-based imputation.
  • Goal: Complete dataset so ML algorithms can work effectively.

  • ML models need numeric input → categorical values must be encoded.
  • Methods:
    • Label Encoding: Assign numbers (red=0, green=1, blue=2). Good for ordinal, risky for nominal.
    • One-Hot Encoding: Create binary columns (red → [1,0,0], green → [0,1,0], blue → [0,0,1]).
    • Target Encoding: Replace categories with target mean (good for high-cardinality).

  • Different features may have different scales (e.g., age=20–60 vs income=30k–150k).
  • Scaling is essential for KNN, SVM, gradient-descent models.
  • Normalization (Min-Max Scaling):
    • Scales values to [0,1].
    • Formula: x' = (x - xmin) / (xmax - xmin)
    • Best when bounded features are needed.
  • Standardization (Z-score):
    • Centers around mean 0, std 1.
    • Formula: x' = (x - μ) / σ
    • Good for Gaussian-like distributions.

  • Feature Engineering: Create new features.
    • From "DOB" → "Age".
    • Combine features (BMI = weight/height²).
  • Feature Selection: Keep the most important features.
    • Filter: Correlation, chi-square test.
    • Wrapper: Forward/backward selection, RFE.
    • Embedded: Feature importance (Random Forest, Lasso).

In short: Handle missing data → Encode categoricals → Scale → Engineer & select features.


5. Model Evaluation & Metrics

Progress:

  • Accuracy: (TP + TN) / (TP + TN + FP + FN) → unreliable with imbalanced data.
  • Precision: TP / (TP + FP) → focus on false positives (e.g., spam).
  • Recall (Sensitivity): TP / (TP + FN) → focus on false negatives (e.g., disease detection).
  • F1 Score: 2 * (Precision * Recall) / (Precision + Recall) → balance between precision & recall.

  • ROC Curve: Plots TPR (Recall) vs. FPR at different thresholds.
  • AUC: Measures discrimination ability.
    • 0.5 → Random guessing
    • 1.0 → Perfect classification

                           Predicted
                         Positive   Negative
              Actual +    TP          FN
              Actual -    FP          TN

Helps visualize false positives vs. false negatives.

  • MSE: (1/n) Σ (yᵢ - ŷᵢ)² → average squared error.
  • RMSE: √MSE → interpretable in original units.
  • R²: 1 - (SS_res / SS_tot) → variance explained by model (closer to 1 = better).

  • Cross-validation: Splits data into folds for stable evaluation.
    • k-fold: Train on k-1 folds, test on 1 fold, repeat.
  • Hyperparameter Tuning:
    • Grid Search → exhaustive.
    • Random Search → sample random combinations.
    • Bayesian / Hyperopt / Optuna → efficient, fewer trials.

7. ML in Practice

Progress:

  • Why pipelines?: Automate workflows → preprocessing → training → evaluation → deployment.
  • Scikit-learn Pipelines: Combine preprocessing (scaling, encoding) + estimator into one workflow.
  • Advantages: Reproducibility, reduced data leakage, easier experimentation.

  • Grid Search: Exhaustive search over parameter grid (e.g., learning rate = [0.01, 0.1, 1]).
  • Random Search: Randomly samples parameter combinations → faster than grid search.
  • Bayesian Optimization: Models objective function, selects next best hyperparameters (Optuna, Hyperopt).
  • Best Practice: Always combine with cross-validation.

  • REST API: Flask/FastAPI → expose model as API endpoint.
  • Example: /predict endpoint takes JSON input, returns predictions.
  • Serialization: Save trained model (joblib, pickle, torch.save).
  • Cloud Deployment: AWS (SageMaker, Lambda, EC2), GCP (Vertex AI), Azure ML.
  • Scaling: Containerization with Docker + deployment via Kubernetes.

  • Oversampling: Duplicate minority class samples (risk: overfitting).
  • Undersampling: Reduce majority class samples (risk: losing information).
  • SMOTE: Generate synthetic minority samples → more balanced.
  • Algorithmic Methods: Use class weights (class_weight='balanced' in scikit-learn, focal loss in DL).

8. AI/ML System Design Concepts

Progress:

  • Collaborative Filtering: Based on user-user or item-item similarity. Example: “Users who liked X also liked Y.”
  • Content-Based Filtering: Uses item features (genre, tags, description).
  • Hybrid Approaches: Combine collaborative + content-based to overcome limitations.
  • Scaling Challenges: Cold start problem, real-time recommendations (low latency), distributed matrix factorization (Spark, GPUs).

  • Use Cases: Fraud detection, recommendation, ad targeting, autonomous driving.
  • System Requirements: Low-latency predictions (ms), high throughput (thousands/millions req/s).
  • Architecture: Model served via REST API / gRPC, inference-optimized runtimes (TensorRT, ONNX Runtime), cache frequent results (Redis).
  • Scaling: Auto-scaling with Kubernetes, message queues (Kafka, RabbitMQ) for async workloads.

  • Batch Processing: Large datasets processed periodically (daily/weekly). Tools: Hadoop, Spark. Use case: training recommendation models.
  • Streaming Processing: Continuous, real-time ingestion & processing. Tools: Kafka, Flink, Spark Streaming. Use case: fraud detection, live dashboards.
  • Lambda Architecture:
    • Batch layer: Stores historical data.
    • Speed layer: Processes real-time events.
    • Serving layer: Merges both.

  • Challenges: Large models (billions of parameters), high memory & compute demand.
  • Techniques:
    • Model Parallelism: Split model across GPUs.
    • Data Parallelism: Split training data across nodes.
    • Distillation: Compress large models into smaller ones.
    • Quantization & Pruning: Reduce precision (FP32 → FP16/INT8).
  • Deployment Strategies: Use cloud ML platforms (SageMaker, Vertex AI), load balancing across inference servers, edge deployment for low-latency.

9. Ethics & Responsible AI

Progress:

  • Problem: Models inherit biases from training data → unfair outcomes. Example: hiring favoring one gender.
  • Types of Bias:
    • Sampling bias: Data not representative.
    • Measurement bias: Features/labels inaccurately recorded.
    • Algorithmic bias: Model amplifies existing biases.
  • Mitigation: Diverse datasets, fairness-aware algorithms (reweighting, adversarial debiasing), regular audits.

  • Risks: Data leaks, re-identification, model inversion attacks.
  • Best Practices:
    • Data anonymization & pseudonymization.
    • Differential privacy: add noise to protect individuals.
    • Secure storage, access controls, encryption in transit and at rest.

  • Purpose: Understand why a model made a prediction; improves trust, compliance, debugging.
  • Techniques:
    • SHAP / LIME → feature importance explanations.
    • Attention visualization → NLP/vision models.
    • Rule extraction / surrogate models → simpler interpretable models mimicking complex ones.
