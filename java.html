Nerd-Nest

1. Java Basics

  • Popular, platform-independent, object-oriented programming language.
  • Write once, run anywhere (thanks to JVM).
  • Strongly typed and compiled to bytecode.

  • Created by Sun Microsystems in 1995.
  • Features:
    • Object-oriented
    • Platform-independent (JVM)
    • Automatic memory management (Garbage Collection)
    • Multi-threading support
    • Rich standard library
      •

  • JVM (Java Virtual Machine): Runs Java bytecode on any platform.
  • JRE (Java Runtime Environment): JVM + libraries needed to run Java apps.
  • JDK (Java Development Kit): JRE + tools to develop Java apps (compiler, debugger).

  • Java source code (.java) → compiled by javac → bytecode (.class) → run by JVM.

  • Structure:
    public class Main {
  public static void main(String[] args) {
      System.out.println("Hello, world!");
  }
}

  • Primitive types: int, byte, short, long, float, double, char, boolean.
  • Variables: Named storage for data, must declare type.
  • Literals: Fixed values in code, e.g., 10, 'A', true, "hello".

  • Arithmetic: +, -, *, /, %
  • Relational: ==, !=, <, >, <=, >=
  • Logical: &&, ||, !
  • Bitwise: &, |, ^, ~, <<, >>, >>>
  • Assignment: =, +=, -=, etc.
  • Unary: ++, --, +, -, !
  • Ternary: condition ? expr1 : expr2

  • If, Else If, Switch
    • if checks condition, executes block if true.
    • else if for multiple conditions.
    • switch matches value with cases, good for discrete values.
    • switch(day) {
  case 1: System.out.println("Monday"); break;
  default: System.out.println("Other day");
}
  • Loops: For, While, Do-While, Enhanced For
    • for: Loop with init, condition, update.
    • while: Loops while condition is true.
    • do-while: Executes once, then repeats while condition true.
    • Enhanced For: For iterating collections/arrays easily.
    • for(String s : list) { 
  System.out.println(s); 
}
  • Break, Continue, Return
    • break: Exits loop or switch.
    • continue: Skips current iteration, goes to next.
    • return: Exits method and optionally returns value.

  • Scanner: Simplest way to read input from console. 
    Scanner sc = new Scanner(System.in);
int num = sc.nextInt();
  • BufferedReader: Older, faster way to read input line-by-line. 
    BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
String line = br.readLine();
  • Command-Line Arguments: Passed as String[] args in main method, used to pass inputs when starting program.

  • Printing with System.out and printf:
    • System.out.println("Hello") — prints with newline.
    • System.out.print("Hello") — prints without newline.
    • System.out.printf("Hello %s", name) — formatted printing (like C’s printf).

2. OOP Principles

  • Core concepts:
    • Encapsulation
    • Inheritance
    • Polymorphism
    • Abstraction
      •
  • These principles help design modular, reusable, and maintainable code.

  • Wrapping data (variables) and methods that operate on data into a single unit (class).
  • Use private fields and provide public getters/setters to control access.
  • Benefits: Data hiding, protects object state.
  • Ex: A capsule which is a collection of several medicines.

  • Mechanism where one class (child/subclass) inherits fields and methods of another (parent/superclass).
  • Types:
    • One parent, one child.
    • Multilevel: Chain of inheritance (grandparent → parent → child).
    • Hierarchical: One parent, multiple children.
  • Syntax: 
    class Parent { ... }
class Child extends Parent { ... }
  • Ex: Cars are inherited from automobiles.

  • Ability to take many forms.
  • Types:
    • Compile-time polymorphism (Method Overloading): Same method name, different parameters.
    • Runtime polymorphism (Method Overriding): Child class provides specific implementation of a method defined in parent.
  • Example Overloading: 
    void print(int a) { }
void print(String s) { }
  • Example Overriding: 
    @Override
void print() { System.out.println("Child"); }
  • Ex: A person can be employee at office, father at home, a player at playground.

  • Hiding internal details and showing only essential features.
  • Implemented using:
    • Abstract Classes: Can have abstract (no body) and concrete methods. Cannot be instantiated directly.
    • Interfaces: Only method signatures (Java 8+ allows default methods).
    • Sealed Classes (Java 15+): Restrict which classes can extend a class.
  • Example: 
    abstract class Animal { abstract void sound(); }
interface Flyable { void fly(); }
  • Ex: Receive cash using ATM card. Internal process is hidden, only receive cash via ATM.

  • Class: Blueprint/template for creating objects.
  • Object: Instance of a class with actual data.

  • Special methods to initialize objects.
  • Types:
    • Default: No-argument constructor (auto-provided if none defined).
    • Parameterized: Accept arguments to initialize fields.
    • Copy constructor: Create new object as a copy of existing one (not built-in in Java but can be defined).
  • Example: 
    public class Book {
  public Book() { }  // Default
  public Book(String title) { }  // Parameterized
}

  • this → Refers to current object instance. Used to resolve naming conflicts or call other constructors.
  • super → Refers to parent class. Used to call parent constructor or overridden methods.

  • public: Accessible everywhere.
  • private: Accessible only within the class.
  • protected: Accessible within package and subclasses.
  • default (package-private): Accessible only within the package (no modifier).

  • Static Nested Class: Defined with static keyword inside another class. Does not have access to outer class’s instance members.
  • Non-static Inner Class: Tied to instance of outer class. Has access to outer instance members.
  • Local Inner Class: Defined inside a method. Limited to method scope.
  • Anonymous Inner Class: No class name; defined and instantiated in one expression, typically for implementing interfaces or extending classes on the fly.

3. Language Features

  • Makes a member (variable, method, block) belong to the class rather than a specific object.
    • Static variables: Shared among all objects of the class.
      Example: static int count; — one count for all instances.
    • Static methods: Can be called without creating an object.
      Example: Math.max(3, 5) — max is static.
    • Static blocks: Run once when the class is loaded, usually for initializing static data.
      Example: 
      static {
    System.out.println("Class loaded!");
}

  • Means cannot be changed (in different contexts):
    • Final variables: Once assigned, value cannot change (constant).
      Example: final int x = 10;
    • Final methods: Cannot be overridden in subclasses.
      Example: 
      final void show() { }
    • Final classes: Cannot be extended.
      Example: public final class String { ... }

  • Enums: Represent a fixed set of constants.
    • Example: 
      enum Day { MONDAY, TUESDAY, WEDNESDAY }
    • Use for predefined values like days, directions, status codes.

  • Records: Immutable data classes that store values without boilerplate code.
    • Example: 
      record Person(String name, int age) {}
    • Automatically provides constructor, getters, equals(), hashCode(), and toString().

  • Allows local variable type inference — Java figures out the type from the value assigned.
  • Works only for local variables (inside methods, loops, etc.), not fields or method parameters.
  • Example: 
    var name = "John"; // String
var age = 25;      // int
  • Improves readability when the type is obvious.

  • Restrict which classes can extend or implement them. Improves security and control over class hierarchy.
  • Example: 
    public sealed class Shape permits Circle, Square {}
public final class Circle extends Shape {}
public final class Square extends Shape {}
  • Only Circle and Square can extend Shape.

4. Collections Framework

  • Provides a standard set of interfaces to store and manipulate groups of objects:
    • List – Ordered collection, allows duplicates.
    • Set – Unordered collection, no duplicates.
    • Queue – FIFO (First-In-First-Out) structure, for processing in order.
    • Deque – Double-ended queue, allows adding/removing at both ends.
    • Map – Key-value pairs (keys are unique, values can repeat).

  • List Implementations:
    • ArrayList – Dynamic array, fast random access, slower insert/remove in middle.
    • LinkedList – Doubly linked list, faster insert/remove in middle, slower random access.
  • Set Implementations:
    • HashSet – Unordered, uses hashing for fast lookups.
    • LinkedHashSet – Maintains insertion order.
    • TreeSet – Sorted set using natural order or custom comparator.
  • Map Implementations:
    • HashMap – Unordered, fast lookups via hashing.
    • LinkedHashMap – Maintains insertion order.
    • TreeMap – Sorted map by keys.
    • ConcurrentHashMap – Thread-safe, high-performance map for concurrent access.

  • Collections – Static methods for operations like sorting, searching, reversing, synchronizing, creating unmodifiable collections.
  • Arrays – Static methods for array operations like sorting, searching, copying, filling.

  • Comparable – Natural ordering of objects. Implemented in the class itself via compareTo().
  • Comparator – External ordering logic, can define multiple sorting criteria using compare().

  • Iterator – Traverse collections forward only, can remove elements during iteration.
  • ListIterator – Traverse lists both forward and backward.
  • For-each loop – Simplified iteration, cannot remove elements.

  • Created using List.of(), Set.of(), Map.of() methods.
  • Cannot be modified after creation (add/remove/update operations throw exceptions).
  • Useful for read-only data.

  • List:
    Implementation Order Duplicates Nulls Access Use Case
    ArrayList Maintains insertion order Allowed Multiple Index-based (fast random access) Playlist, dynamic arrays
    LinkedList Maintains insertion order Allowed Multiple Sequential access, fast insert/delete Queue, frequent insert/delete
    Vector Maintains insertion order Allowed Multiple Index-based, synchronized Legacy, thread-safe list
    Stack LIFO order Allowed Multiple Top element access Undo operations, expression evaluation
  • Set:
    Implementation Order Duplicates Nulls Access Use Case
    HashSet No guaranteed order Not allowed One null allowed By element only Unique IDs, membership check
    LinkedHashSet Maintains insertion order Not allowed One null allowed By element only Caches, preserve order
    TreeSet Sorted (natural/comparator) Not allowed Not allowed By element only (sorted access) Sorted unique data, leaderboards
  • Queue:
    Implementation Order Duplicates Nulls Access Use Case
    Queue/Deque FIFO / Both ends Allowed One null allowed Head / Tail access Task scheduling, double-ended queue
    PriorityQueue Priority order Allowed Not allowed Head (highest priority) Job scheduling, task prioritization
    ArrayDeque FIFO / LIFO (Deque) Allowed Not allowed Both ends Stack/Queue replacement
  • Map:
    Implementation Order Duplicate Keys Duplicate Values Nulls Use Case
    HashMap No guaranteed order Not allowed Allowed One null key, multiple null values General key-value mapping
    LinkedHashMap Maintains insertion order Not allowed Allowed One null key, multiple null values Caches, preserve order
    TreeMap Sorted by keys Not allowed Allowed No null keys, null values allowed Sorted dictionary, range queries
    Hashtable No guaranteed order Not allowed Allowed No null keys/values Legacy, thread-safe map

5. Exception Handling

  • try – Block of code that might throw an exception.
  • catch – Handles the exception if it occurs.
  • finally – Always runs after try/catch, often used to close resources.
  • Example:
    try {
    int result = 10 / 0; // Throws ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero");
} finally {
    System.out.println("This will always run");
}

  • throw – Actually throws an exception object.
  • throws – Declares that a method may throw exceptions.
  • Example:
    void checkAge(int age) throws Exception { // 'throws' declaration
    if (age < 18) {
        throw new Exception("Underage not allowed"); // 'throw' usage
    }
}

  • Checked – Checked at compile time; must be handled or declared (e.g., IOException).
  • Unchecked – Checked at runtime; usually programming errors (e.g., NullPointerException).
  • Example:
    // Checked
FileReader fr = new FileReader("file.txt"); // Must handle IOException
                                          
// Unchecked
String s = null;
System.out.println(s.length()); // NullPointerException at runtime

  • You can create your own exception classes for specific scenarios.
  • Extend Exception for checked exceptions, RuntimeException for unchecked.
  • Example:
    class InvalidInputException extends Exception {
    public InvalidInputException(String message) {
        super(message);
    }
}
                                        
void validate(int value) throws InvalidInputException {
    if (value < 0) throw new InvalidInputException("Negative not allowed");
}

  • Allows catching multiple exception types in a single catch using |.
  • Example:
    try {
  String text = null;
  System.out.println(text.length());
} catch (NullPointerException | ArithmeticException e) {
  System.out.println("Caught exception: " + e.getClass().getSimpleName());
}

  • Automatically closes resources (files, DB connections) after use.
  • Example:
    try (FileReader fr = new FileReader("data.txt");
    BufferedReader br = new BufferedReader(fr)) {
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
}

6. I/O & File Handling

  • APIs to read/write data from different sources (files, network, memory, etc.). These APIs are mainly in:
    • java.io package → Traditional I/O (byte streams & character streams)
    • java.nio and java.nio.file → New I/O with better performance and features (introduced in Java 1.4 & enhanced in Java 7)

  • Used for binary data like images, audio, PDFs.
    • InputStream → Reads bytes from a source.
    • OutputStream → Writes bytes to a destination.
    • Deals with raw bytes, not characters.
  • Example: Copy a binary file 
    import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.IOException;
            
public class ByteStreamExample {
    public static void main(String[] args) {
        try (FileInputStream in = new FileInputStream("input.jpg");
              FileOutputStream out = new FileOutputStream("output.jpg")) {
            
            int byteData;
            while ((byteData = in.read()) != -1) {
                out.write(byteData);
            }
            System.out.println("File copied successfully.");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
  • When to use: Binary files (images, videos, audio).

  • Used for text data (characters). Automatically handles character encoding (UTF-8, etc.).
    • Reader → Reads characters.
    • Writer → Writes characters.
  • Example:
    import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;
                                      
public class CharacterStreamExample {
    public static void main(String[] args) {
        // Write to file
        try (FileWriter writer = new FileWriter("data.txt")) {
            writer.write("Hello Java I/O");
        } catch (IOException e) {
            e.printStackTrace();
        }
                                      
        // Read from file
        try (FileReader reader = new FileReader("data.txt")) {
            int ch;
            while ((ch = reader.read()) != -1) {
                System.out.print((char) ch);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
  • When to use: Plain text files.

  • Wraps around character or byte streams to improve performance by reading/writing chunks instead of single characters/bytes.
    • BufferedReader → Efficient reading (also provides readLine()).
    • BufferedWriter → Efficient writing.
  • Example:
    import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;
                                      
public class BufferedStreamExample {
    public static void main(String[] args) {
        try (BufferedReader br = new BufferedReader(new FileReader("data.txt"))) {
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
  • When to use: Large text files or frequent read/write operations.

  • Represents file/directory pathnames, not file content. Can create, delete, rename, and check file properties.
  • Example: 
    import java.io.File;
import java.io.IOException;
                                      
public class FileAPIExample {
    public static void main(String[] args) {
        File file = new File("example.txt");
                                      
        try {
            if (file.createNewFile()) {
                  System.out.println("File created: " + file.getName());
            } else {
                System.out.println("File already exists.");
            }
                                      
            System.out.println("Path: " + file.getAbsolutePath());
            System.out.println("Writable: " + file.canWrite());
            System.out.println("Readable: " + file.canRead());
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

  • When to use: Managing file/directory metadata.

  • Modern and efficient file handling introduced in Java 1.4, with enhancements in Java 7.
    • Path → Modern file path replacement for File.
    • Files utility class → Easy read/write/copy/move/delete operations.
  • Example:
    import java.nio.file.*;
                                      
public class NIOExample {
    public static void main(String[] args) {
        Path path = Paths.get("nio_example.txt");
                                      
        try {
            // Write to file
            Files.write(path, "Hello NIO.2".getBytes());
                                      
            // Read file
            String content = Files.readString(path);
            System.out.println(content);
                                      
            // Copy file
            Files.copy(path, Paths.get("nio_copy.txt"), StandardCopyOption.REPLACE_EXISTING);
        } catch (Exception e) {
             e.printStackTrace();
        }
    }
}
  • When to use: Modern and efficient file handling with more utilities than java.io.File.

7. Generics

  • Generics let you parameterize types (classes, interfaces, methods) so you get compile-time type safety and reusable code without casts.

  • Generic classes take type parameters (e.g. Box<T>). Generic methods declare their own type parameters and can be used even in non-generic classes.
  • Reuse code with different types while avoiding casts.
  • Example: Generic Class
    public class Box<T> {
    private T value;
    public Box(T value) { this.value = value; }
    public T get() { return value; }
    public void set(T v) { value = v; }
}
                                      
// usage
Box<String> b = new Box<>("hello");
String s = b.get(); // no cast
  • Example: Generic Method
    public class Utils {
    public static <T> T pickFirst(T[] arr) {
        return arr.length == 0 ? null : arr[0];
    }
                                      
    // bounded generic method
    public static <T extends Comparable<T>> T max(T a, T b) {
        return a.compareTo(b) >= 0 ? a : b;
    }
}
                                      
// usage
Integer[] nums = {1, 2, 3};
Integer first = Utils.pickFirst(nums);
Integer m = Utils.max(4, 7);
  • Note: Method type parameters (<T>) are declared before the return type.

  • Restrict type parameters with extends so you can use methods of that bound.
  • Syntax: <T extends SomeClassOrInterface>
  • You can have multiple bounds: <T extends ClassA & InterfaceB> (class bound must come first).
  • Example:
    // Upper bound with Number
public class NumericBox<T extends Number> {
    private T value;
    public NumericBox(T v) { value = v; }
    public double doubleValue() { return value.doubleValue(); }
}
                                      
// Multiple bounds: class + interface
public class Bounded<T extends Number & Comparable<T>> {
    public boolean greater(T a, T b) {
        return a.compareTo(b) > 0;
     }
}
  • Rule: extends is also used for interfaces in bounds (it means “is-a” or “implements”).

  • Wildcards make APIs flexible by allowing "some unknown type" rather than a concrete type parameter.
    • List<?> — list of some unknown type (read-only except null).
    • List<? extends T> — list of some subtype of T (read as T, cannot add T).
    • List<? super T> — list of some supertype of T (can add T, reading yields Object or the supertype).
  • PECS Rule: Producer Extends, Consumer Super.
  • Example: 
    List<Integer> ints = List.of(1,2,3);
List<? extends Number> nums = ints;
Number n = nums.get(0); // OK
// nums.add(10); // ERROR
                                      
List<Object> objects = new ArrayList<>();
List<? super Integer> ints2 = objects;
ints2.add(10);          // OK
Object o = ints2.get(0);// yields Object
                                      
// Copy method using PECS
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (T item : src) dest.add(item);
}
  • Avoid: Raw types (e.g. List) — they disable generics checks and cause unsafe casts.

  • Java generics are implemented via type erasure — type parameters exist only at compile time.
    • List<String> and List<Integer> are the same class at runtime.
    • Cannot do new T[] or instanceof List<String>.
    • Cannot create new T().
    • Static members cannot use the class type parameter.
  • Example:
    List<String> ls = new ArrayList<>();
List<Integer> li = new ArrayList<>();
System.out.println(ls.getClass() == li.getClass()); // true
                                      
// Illegal:
// T[] arr = new T[10];
// if (obj instanceof List<String>) {}
  • Why: Backwards compatibility — generics were added after many APIs existed, so erasure allowed old and new code to work together.

8. Concurrency & Multithreading

  • A thread goes through these states (from Thread.State enum):
    StateMeaning
    NEWThread object created, not started yet (new Thread()).
    RUNNABLEReady to run, waiting for CPU.
    RUNNING*(Not actual enum) Part of RUNNABLE, actually executing.
    BLOCKEDWaiting to acquire a lock.
    WAITINGWaiting indefinitely for another thread’s signal (wait()).
    TIMED_WAITINGWaiting for a fixed time (sleep(ms), join(ms)).
    TERMINATEDThread finished execution.
  • Example: 
    Thread t = new Thread(() -> System.out.println("Running"));
System.out.println(t.getState()); // NEW
t.start();
System.out.println(t.getState()); // RUNNABLE or RUNNING

  • Using Thread class: 
    class MyThread extends Thread {
    public void run() { System.out.println("Thread running"); }
}
new MyThread().start();
  • Using Runnable interface (preferred): 
    Runnable task = () -> System.out.println("Runnable running");
new Thread(task).start();
  • Why Runnable is better: Allows extending other classes and separates task from execution thread.

  • Prevents race conditions by ensuring only one thread accesses a block at a time.
  • Synchronized method: 
    synchronized void increment() { 
  count++; 
}
  • Synchronized block: 
    void increment() {
    synchronized(this) { 
      count++;
    }
}
  • Explicit Locks (ReentrantLock): 
    import java.util.concurrent.locks.ReentrantLock;

ReentrantLock lock = new ReentrantLock();
lock.lock();

try { 
  count++; 
} finally { 
  lock.unlock(); 
}
  • Best practice: Use locks only where needed to avoid performance bottlenecks.

  • Ensures visibility of changes to a variable across threads. Without volatile, threads may read cached values. Does NOT provide atomicity.
  • Example: 
    volatile boolean running = true;
            
public void run() {
    while (running) { }
}
  • If another thread sets running = false;, this change is immediately visible.

  • Creating many threads manually is costly. Executors manage thread pools to reuse threads efficiently.
  • Example: 
    import java.util.concurrent.*;
            
ExecutorService executor = Executors.newFixedThreadPool(3);
executor.submit(() -> System.out.println("Task"));
executor.shutdown();
  • Common factory methods:
    • newFixedThreadPool(n) — fixed number of threads
    • newCachedThreadPool() — expands/shrinks dynamically
    • newSingleThreadExecutor() — single thread only

  • Runnable returns nothing and can't throw checked exceptions.
  • Callable<T> returns a value and can throw exceptions.
  • Future<T> represents the result of async computation.
  • Example: 
    Callable<Integer> task = () -> 5 + 5;
ExecutorService ex = Executors.newSingleThreadExecutor();
Future<Integer> future = ex.submit(task);
System.out.println(future.get()); // blocks until result ready
ex.shutdown();

  • Specialized pool for divide-and-conquer parallelism using work-stealing. Uses RecursiveTask<V> (returns result) or RecursiveAction (no result).
  • Example: 
    import java.util.concurrent.*;
            
class SumTask extends RecursiveTask<Integer> {
    int[] arr; int start, end;
    SumTask(int[] arr, int start, int end) { this.arr = arr; this.start = start; this.end = end; }
    protected Integer compute() {
        if (end - start <= 2) {
            int sum = 0; for (int i = start; i < end; i++) sum += arr[i];
            return sum;
        }
        int mid = (start + end) / 2;
        SumTask left = new SumTask(arr, start, mid);
        SumTask right = new SumTask(arr, mid, end);
        left.fork();
        return right.compute() + left.join();
    }
}
            
ForkJoinPool pool = new ForkJoinPool();
int sum = pool.invoke(new SumTask(new int[]{1,2,3,4}, 0, 4));
System.out.println(sum);

  • Asynchronous computation with chaining and combining results without blocking.
  • Example: 
    import java.util.concurrent.*;
            
CompletableFuture.supplyAsync(() -> "Hello")
    .thenApply(str -> str + " World")
    .thenAccept(System.out::println); // prints Hello World
  • Features:
    • Non-blocking async execution.
    • Combine multiple futures: thenCombine(), allOf(), anyOf().

  • Java 8+ streams can run in parallel to utilize multiple CPU cores.
  • Example: 
    import java.util.stream.IntStream;
            
int sum = IntStream.range(1, 1000)
                    .parallel()
                    .sum();
System.out.println(sum);
  • Notes:
    • Internally uses ForkJoinPool.
    • Good for CPU-bound tasks but be cautious with shared mutable state.

9. Functional Programming

  • Anonymous functions (functions without a name).
  • Used to provide implementations of functional interfaces (interfaces with a single abstract method).
  • Syntax: (parameters) -> expression or (parameters) -> { statements; }
  • Example:
    • Runnable r = () -> System.out.println("Hello from lambda");
r.run();
  • Lambdas replace verbose anonymous inner classes.

  • Interfaces with only one abstract method.
  • Optionally marked with @FunctionalInterface annotation to help compiler checks.
  • Examples: Runnable, Callable<T>, Comparator<T>, and custom ones.
  • Example:
    • @FunctionalInterface
interface MyFunc {
    void execute(String s);
}
            
MyFunc func = s -> System.out.println(s.toUpperCase());
func.execute("hello");

  • Shorter way to write lambdas calling existing methods.
  • Syntax:
    • ClassName::staticMethod
    • instance::instanceMethod
    • ClassName::instanceMethod (for instance methods on parameters)
  • Example:
    • List<String> list = List.of("a", "b", "c");
list.forEach(System.out::println); // method reference instead of lambda (s -> System.out.println(s))

  • Streams represent sequences of data with operations that can be lazy and chained.
  • Intermediate operations: return a stream (e.g. filter(), map()) and are lazy.
  • Terminal operations: produce a result or side-effect (e.g. collect(), forEach(), count()) and trigger execution.
  • Example:
    • List<String> names = List.of("John", "Jane", "Jack");
long count = names.stream()
                  .filter(s -> s.startsWith("J"))  // intermediate
                  .count();                        // terminal
System.out.println(count); // 3

  • Container for optional (possibly missing) values.
  • Avoids NullPointerException and explicit null checks.
  • Common methods: of(), ofNullable(), isPresent(), orElse(), ifPresent().

11. Memory Management

Memory Type Purpose Characteristics
Heap Stores objects and instance variables Shared among all threads, larger, managed by GC
Stack Stores method calls, local variables, and references Each thread has its own stack, stores primitive variables and references, follows LIFO
  • Stack: Fast, stores method calls & primitives like int, double, and references to objects.
  • Heap: Stores actual objects created by new. GC cleans unused objects here.
  • Example: 
    public class MemoryExample {
    public static void main(String[] args) {
        int x = 10;               // stored in stack
        String str = new String("Hello");  // str reference in stack, "Hello" object in heap
    }
}

  • Java automatically frees heap memory for objects no longer referenced.
  • Garbage Collector (GC) runs periodically.
  • GC algorithms include mark-and-sweep, generational, etc.
  • You can suggest GC with System.gc() (not guaranteed).
  • What GC does:
    • Finds objects no longer reachable.
    • Frees their memory.

Reference Type Behavior Use Case
Strong Default reference; object won’t be collected if referenced Normal objects
Soft GC collects if memory is low Caches
Weak GC collects when no strong references exist WeakHashMap keys
Phantom Used to schedule cleanup after GC collects object Advanced cleanup, finalize alternatives
  • Example: WeakReference 
    import java.lang.ref.WeakReference;
                
    public class WeakRefExample {
        public static void main(String[] args) {
            String strong = new String("Strong");
            WeakReference<String> weak = new WeakReference<>(strong);
                
            strong = null;  // now only weak references the object
            System.gc();
                
            System.out.println("Weak reference: " + weak.get()); // may print null if GC collected
        }
    }

  • Finalization (finalize() method) was used to clean resources before GC but is deprecated and unpredictable.
  • Cleaner API (Java 9+) is a better alternative for resource cleanup.
  • Cleaner example:
    • import java.lang.ref.Cleaner;
            
public class CleanerExample {
    static class State implements Runnable {
        public void run() {
            System.out.println("Cleaning resources...");
        }
    }
            
    static class Resource {
        private static final Cleaner cleaner = Cleaner.create();
        private final Cleaner.Cleanable cleanable;
            
        Resource() {
            State state = new State();
            cleanable = cleaner.register(this, state);
        }
            
        public void close() {
            cleanable.clean();
        }
    }
            
    public static void main(String[] args) {
        Resource r = new Resource();
        r.close();  // manually trigger cleaning
    }
}

12. Networking

  • Socket: Endpoint for communication between two machines.
  • Java supports two main protocols:
    • TCP (Transmission Control Protocol): Connection-oriented, reliable, stream-based.
    • UDP (User Datagram Protocol): Connectionless, unreliable, packet-based, faster.
  • TCP Socket Example (Client & Server) Server: 
    import java.net.*;
import java.io.*;
            
public class TCPServer {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(1234);
        System.out.println("Server waiting for connection...");
        Socket clientSocket = serverSocket.accept();
        BufferedReader in = new BufferedReader(new InputStreamReader(clientSocket.getInputStream()));
        String message = in.readLine();
        System.out.println("Received: " + message);
        serverSocket.close();
    }
}
    Client: 
    import java.net.*;
import java.io.*;
            
public class TCPClient {
    public static void main(String[] args) throws IOException {
        Socket socket = new Socket("localhost", 1234);
        PrintWriter out = new PrintWriter(socket.getOutputStream(), true);
        out.println("Hello Server");
        socket.close();
    }
}
    UDP Example (Sender & Receiver) Receiver: 
    import java.net.*;
            
public class UDPReceiver {
    public static void main(String[] args) throws Exception {
        DatagramSocket socket = new DatagramSocket(9876);
        byte[] buffer = new byte[1024];
        DatagramPacket packet = new DatagramPacket(buffer, buffer.length);
        socket.receive(packet);
        String received = new String(packet.getData(), 0, packet.getLength());
        System.out.println("Received: " + received);
        socket.close();
    }
}
    Sender: 
    import java.net.*;
            
public class UDPSender {
    public static void main(String[] args) throws Exception {
        DatagramSocket socket = new DatagramSocket();
        byte[] buffer = "Hello UDP".getBytes();
        InetAddress address = InetAddress.getByName("localhost");
        DatagramPacket packet = new DatagramPacket(buffer, buffer.length, address, 9876);
        socket.send(packet);
        socket.close();
    }
}

13. Best Practices

  • Classes & Interfaces: Use PascalCase (e.g., MyClass, Runnable).
  • Methods & Variables: Use camelCase (e.g., calculateSum(), totalCount).
  • Constants: Use UPPER_SNAKE_CASE (e.g., MAX_SIZE).
  • Indentation: Use 4 spaces per level, no tabs.
  • Braces: Use K&R style (opening brace on the same line).
  • Write meaningful, descriptive names.
  • Keep methods short and focused on one task.
  • Example: 
    // Good naming
    public class UserAccount {
        private String userName;
                
        public void calculateBalance() {
            // logic
        }
    }

  • Make objects immutable when possible for thread safety and simplicity.
    • Use final fields, no setters.
    • Create new objects on changes instead of modifying existing ones.
  • Example: 
    public final class ImmutableUser {
    private final String name;
    private final int age;
            
    public ImmutableUser(String name, int age) {
        this.name = name;
        this.age = age;
    }
            
    public String getName() { return name; }
    public int getAge() { return age; }
}

  • Protect internal state when passing or returning mutable objects.
  • Return copies to prevent external modification.
  • Example: 
    import java.util.*;
            
public class SafeClass {
    private final List list;
            
    public SafeClass(List input) {
        this.list = new ArrayList<>(input);  // defensive copy
    }
            
    public List getList() {
        return new ArrayList<>(list);        // return copy
    }
}

  • Nullify references if object is no longer needed and lifecycle is long.
  • Be careful with static collections holding references.
  • Use weak references or proper caches if applicable.
  • Close resources like files, sockets, database connections.
  • Avoid inner classes holding implicit references to outer classes unnecessarily.

  • Choose data structures based on usage:
    • Use ArrayList for fast random access.
    • Use LinkedList for fast insertions/removals.
    • Use HashMap for fast key-based lookups.
    • Use TreeMap when sorted order is required.
    • Avoid overusing collections; pick the simplest one that meets the need.

  • Use checked exceptions for recoverable conditions.
  • Use unchecked exceptions (RuntimeException) for programming errors.
  • Always catch the most specific exception you can.
  • Avoid empty catch blocks.
  • Log exceptions or rethrow with meaningful messages.
  • Clean up resources in finally block or use try-with-resources.
  • Example: 
    try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();  // or proper logging
}
