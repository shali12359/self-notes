Nerd-Nest

1. JavaScript Fundamentals

  • Purpose: Variables store data values.
  • Main data types:
    • Primitive: string, number, boolean, null, undefined, symbol, bigint
    • Reference: object, array, function, etc.
  • Example:
    • let name = "Alex"; // string
let age = 25;      // number
let isActive = true; // boolean

  • var: function-scoped, hoisted, can be re-declared.
  • let: block-scoped, not hoisted in usable form, reassignable but not re-declarable in same scope.
  • const: block-scoped, cannot be reassigned (but reference-type contents can mutate).
  • Example:
    var x = 1;
let y = 2;
const z = 3;

  • Primitive: stored directly, immutable (e.g., numbers, strings).
  • Reference: stored in heap; variables hold references to objects/arrays/functions.
  • Example:
    let a = 10;
let b = a; // b is copy
b = 20; // a remains 10
                            
let obj1 = { name: "John" };
let obj2 = obj1; // both reference same object
obj2.name = "Jane"; // obj1.name is now "Jane"

  • Conversion (explicit): using Number(), String(), etc.
  • Coercion (implicit): JS converts types automatically during operations.
  • Example: 
    Number("42"); // 42
String(123);    // "123"
                            
"5" + 1; // "51"  (string concatenation)
"5" - 1; // 4     (string coerced to number)

  • Arithmetic: +, -, *, /, %, **
  • Comparison: == (loose), === (strict), <, >, ≤, ≥
  • Logical: &&, ||, !
  • Bitwise: &, |, ^, ~, <<, >>

  • Behaviour: returns the right-hand operand only when left is null or undefined.
  • Example:
    let username = null;
console.log(username ?? "Guest"); // "Guest"

  • Purpose: safely access deep properties without throwing if a part is null or undefined.
  • Example:
    let user = { profile: { name: "Sam" } };
console.log(user.profile?.name);   // "Sam"
console.log(user.contact?.phone);  // undefined (no error)

  • If / Else: conditional branching.
  • Switch: multi-branch control.
  • Loops: for, while, do...while, also for...of (values) and for...in (keys).
  • Example:
    if (x > 10) { /* ... */ } else { /* ... */ }
            
switch(day) {
    case "Mon": /* ... */; break;
    default: /* ... */
}
                            
for (let i = 0; i < 5; i++) { /* ... */ }
while (condition) { /* ... */ }
do { /* ... */ } while (condition);
                            
for (let val of array) { /* values */ }
for (let key in object) { /* keys */ }

  • Declaration: hoisted (function foo(){}).
  • Expression: not hoisted (const f = function(){}).
  • Arrow: lexical this, concise syntax (const f = () => {}).
  • Default params, rest & spread: useful patterns.
  • Example:
    // Declaration (hoisted)
function greet() { return "hi"; }
                            
// Expression (not hoisted)
const greet2 = function() { return "hi"; };
                            
// Arrow
const greet3 = () => "hi";
                            
// Defaults
function sum(a = 0, b = 0) { return a + b; }
                            
// Rest
function total(...nums) { return nums.reduce((a, b) => a + b, 0); }
                            
// Spread
let arr1 = [1, 2];
let arr2 = [...arr1, 3, 4];

  • Scope: global vs local; block vs function.
  • Lexical scope: inner functions access outer variables where they were defined.
  • Closures: functions that remember their lexical environment (useful for data privacy, factory functions).
  • Example: 
    function outer() {
    let count = 0;
    return function inner() {
    count++;
    return count;
    };
}

const counter = outer();
console.log(counter()); // 1
console.log(counter()); // 2

2. Advanced Concepts

  • Definition: JavaScript moves declarations to the top of the scope during compilation.
  • Function declarations: fully hoisted (can be called before definition).
  • var variables: hoisted but initialized as undefined.
  • let / const: hoisted but in Temporal Dead Zone (cannot be accessed before declaration).
  • Example:
    greet(); // "Hi"
function greet() { console.log("Hi"); }
                                          
console.log(x); // undefined
var x = 5;
                                          
console.log(y); // ReferenceError
let y = 10;

  • Execution Context: environment where JS code runs (global, function, eval).
  • Creation phase: hoisting happens, this binding set.
  • Execution phase: code runs line-by-line.
  • Call Stack: tracks function calls (LIFO - last in, first out).
  • Example: 
    function first() { second(); }
function second() { console.log("Hello"); }
first();
// Stack: first → second → pop → pop

  • Single-threaded: JS runs one task at a time.
  • Event loop: checks call stack; if empty, takes tasks from callback/microtask queues.
  • Concurrency model: uses async callbacks, promises, timers.

  • Macrotasks: setTimeout, setInterval, I/O callbacks — run after current stack is clear.
  • Microtasks: Promise.then, queueMicrotask, MutationObserver — run before next macrotask.
  • Example:
    setTimeout(() => console.log("Macrotask"), 0);
Promise.resolve().then(() => console.log("Microtask"));
// Output: Microtask → Macrotask

  • setTimeout(fn, ms): run once after delay.
  • setInterval(fn, ms): run repeatedly.
  • Promise: represents future value (pending → fulfilled/rejected).
  • Example:
    setTimeout(() => console.log("Later"), 1000);
let id = setInterval(() => console.log("Repeat"), 500);
clearInterval(id);
                            
Promise.resolve("Done").then(console.log);

  • Objects inherit from other objects via the prototype chain. Every object has an internal [[Prototype]] link to another object or null.

  • Chain of objects linked through __proto__.
  • Property lookup climbs the chain until found or null.
  • Example: 
    function Person(name) { this.name = name; }
Person.prototype.greet = function() { console.log("Hi " + this.name); };
const p = new Person("Alex");
p.greet(); // via prototype

  • Creates a new object with the specified prototype. 
    const proto = { greet() { console.log("Hello"); } };
const obj = Object.create(proto);
obj.greet();

  • Global scope: this is window (browser) or global (Node).
  • Function (non-strict): this = global; strict mode → undefined.
  • Method: this = object before dot.
  • Arrow function: no own this, inherits from enclosing scope.
  • Class: this refers to the instance.

  • call: invoke with given this and args (comma-separated).
  • apply: same but args as array.
  • bind: returns new function with fixed this.
  • Example: 
    function greet(g) { console.log(g + " " + this.name); }
const person = { name: "Sam" };
greet.call(person, "Hi");
greet.apply(person, ["Hello"]);
const bound = greet.bind(person, "Hey");
bound();

  • Currying: transform multi-arg function into sequence of single-arg functions.
  • Partial application: fix some args and return new function.
  • Example:
    // Currying
const add = a => b => a + b;
add(2)(3); // 5
                            
// Partial application
function multiply(a, b) { return a * b; }
const double = multiply.bind(null, 2);
double(5); // 10

  • Caching function results to avoid recalculation.

    function memoize(fn) {
  const cache = {};
  return function(...args) {
    let key = JSON.stringify(args);
    if (cache[key]) return cache[key];
    cache[key] = fn(...args);
    return cache[key];
  }
}

const slow = n => { for(let i=0;i<1e8;i++); return n; };
const fast = memoize(slow);

3. Object & Array Handling

  • Object literal: 
    const user = { name: "Alex", age: 25 };
  • Constructor function: 
    function User(name, age) { this.name = name; this.age = age; }
              const u = new User("Alex", 25);
  • Object.create(): 
    const proto = { greet() { console.log("Hi"); } };
              const obj = Object.create(proto);
  • Class syntax: 
    class User { constructor(name, age) { this.name = name; this.age = age; } }
  • Factory function: 
    function createUser(name, age) { return { name, age }; }

  • Object destructuring: 
    const person = { name: "Sam", age: 30 };
              const { name, age } = person;
  • Array destructuring: 
    const arr = [1, 2, 3];
              const [a, b] = arr;
  • Renaming & default values: 
    const { name: firstName = "Guest" } = {};
  • Nested destructuring: 
    const user = { info: { city: "NY" } };
              const { info: { city } } = user;

  • Object.keys(obj) → array of property names
  • Object.values(obj) → array of property values
  • Object.entries(obj) → array of [key, value] pairs
  • const obj = { a: 1, b: 2 };
              Object.keys(obj);   // ["a", "b"]
              Object.values(obj); // [1, 2]
              Object.entries(obj);// [["a", 1], ["b", 2]]

  • map: Transforms array elements → new array. 
    [1, 2, 3].map(x => x * 2); // [2, 4, 6]
  • filter: Keeps elements passing condition. 
    [1, 2, 3].filter(x => x > 1); // [2, 3]
  • reduce: Accumulates to single value. 
    [1, 2, 3].reduce((sum, x) => sum + x, 0); // 6
  • forEach: Runs function for each element (no return). 
    [1, 2, 3].forEach(x => console.log(x));
  • find: Returns first match. 
    [1, 2, 3].find(x => x > 1); // 2
  • some: At least one matches. 
    [1, 2, 3].some(x => x > 2); // true
  • every: All must match. 
    [1, 2, 3].every(x => x > 0); // true

  • flat(depth): 
    [1, [2, [3]]].flat(2); // [1, 2, 3]
  • flatMap: 
    ["a", "b"].flatMap(x => [x, x.toUpperCase()]);
              // ["a", "A", "b", "B"]

  • Sorting: 
    [3, 1, 2].sort(); // [1, 2, 3] (lexical sort)
              [3, 1, 2].sort((a, b) => a - b); // numeric sort
  • Searching: 
    [10, 20, 30].includes(20); // true
              [10, 20, 30].indexOf(20);  // 1

5. Asynchronous JavaScript

  • A callback is a function passed as an argument to another function, executed later (often after async work).

    function fetchData(cb) {
  setTimeout(() => cb("Data loaded"), 1000);
}

fetchData(result => console.log(result));

  • Drawback: Callback hell → deeply nested callbacks that are hard to read/manage.

  • A Promise represents a value that will be available in the future.

  • States: pending → fulfilled or rejected.

  • Example: 
    const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve("Done"), 1000);
});

  • .then() → Runs on fulfillment.
  • .catch() → Runs on rejection.
  • .finally() → Runs regardless of outcome.
  • Example:
    p.then(data => console.log(data))
    .catch(err => console.error(err))
    .finally(() => console.log("Finished"));

  • Syntactic sugar over Promises for cleaner code.

  • await pauses inside async function until the Promise resolves/rejects.

  • Example:
    async function load() {
  try {
    const data = await p;
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

load();

  • With try/catch:

    async function getData() {
  try {
    const res = await fetch("bad_url");
    const json = await res.json();
  } catch (error) {
    console.error("Error:", error);
  }
}

  • With .catch():

    fetch("bad_url").catch(err => console.error(err));

  • Promise.all([...]) → Resolves when all succeed, rejects if one fails.
    Promise.all([p1, p2]).then(console.log).catch(console.error);
  • Promise.race([...]) → Resolves/rejects as soon as one settles.
    Promise.race([p1, p2]).then(console.log);
  • Promise.allSettled([...]) → Resolves after all settle.
    Promise.allSettled([p1, p2]).then(console.log);

6. ES6+ and Modern Features

  • Introduced in ES6, enclosed by backticks (` `) instead of quotes.
  • Supports string interpolation (${expression}) and multi-line strings. 
    
let name = "Alex";
let message = `Hello, ${name}! Today is ${new Date().toDateString()}.`;
console.log(message);
                            
// Supports multi-line without \n
let multiline = `Line 1
Line 2`;
  • Why useful: Cleaner syntax, easier to embed variables and expressions.

  • Modules help organize code into separate files. 
    
// math.js
export const add = (a,b) => a + b;
export const sub = (a,b) => a - b;
                            
// main.js
import { add, sub } from './math.js';
                            
// utils.js
export default function log(msg) { console.log(msg); }
                            
// main.js
import log from './utils.js';
  • Why useful: Improves maintainability, avoids global scope pollution.

  • Special functions that can pause and resume execution using yield. 
    
function* countUp() {
  yield 1;
  yield 2;
  yield 3;
}

const counter = countUp();
console.log(counter.next()); // { value: 1, done: false }
  • Why useful: Efficient for sequences, lazy evaluation, async flows.

  • Unique and immutable primitive values, often used as object keys to avoid naming conflicts. 
    
const id = Symbol("id");
const user = { [id]: 123, name: "Sam" };
console.log(user[id]); // 123
  • Why useful: Ensures truly unique property keys, useful for private object properties.

  • Set: Collection of unique values.
  • Map: Key-value pairs where keys can be any type. 
    
const s = new Set([1, 2, 2, 3]);
console.log(s); // Set {1, 2, 3}
                            
const m = new Map();
m.set("name", "Alex");
m.set(1, "Number Key");
console.log(m.get(1));
  • Why useful: More flexible than arrays/objects for certain lookups and uniqueness checks.

  • Safely access nested properties without throwing an error if a value is null or undefined. 
    
let user = { profile: { name: "Sam" } };
console.log(user.profile?.name); // "Sam"
console.log(user.address?.street); // undefined (no error)
  • Why useful: Prevents runtime errors from deeply nested access.

  • Returns right-hand value only if left-hand is null or undefined (unlike ||, which treats 0, "", and false as falsy). 
    
let username = null;
console.log(username ?? "Guest"); // "Guest"
                            
let count = 0;
console.log(count ?? 10); // 0 (keeps 0, unlike ||)
  • Why useful: Provides safer default values without overriding valid falsy values.

7. Object-Oriented Programming (OOP)

  • ES6 introduced class syntax as a cleaner way to create objects and handle inheritance (syntactic sugar over prototypes). 
    
class Person {
  constructor(name) {
    this.name = name;
  }
  greet() {
    console.log(`Hello, my name is ${this.name}`);
  }
}
const p = new Person("Alex");
p.greet(); // Hello, my name is Alex
  • Why useful: Organizes related properties and methods in one blueprint for creating objects.

  • Constructor: Called automatically when creating a new instance.
  • Instance methods: Defined inside the class, accessible to created objects.
  • Static methods: Belong to the class itself, used for utilities. 
    
class MathUtil {
  constructor(num) {
    this.num = num;
  }
  double() {
    return this.num * 2;
  }
  static add(a, b) {
    return a + b;
  }
}

const m = new MathUtil(5);
console.log(m.double()); // 10
console.log(MathUtil.add(3, 4)); // 7
  • Why useful: Separates object-specific behavior from general helper methods.

  • extends creates a subclass from another class.
  • super calls the parent class’s constructor or methods. 
    
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} makes a sound`);
  }
}
class Dog extends Animal {
  constructor(name, breed) {
    super(name);
    this.breed = breed;
  }
  speak() {
    super.speak();
    console.log(`${this.name} barks`);
  }
}
const d = new Dog("Buddy", "Labrador");
d.speak();
  • Why useful: Promotes code reuse and a clear “is-a” relationship between classes.

  • Public fields are accessible anywhere.
  • Private fields (ES2022) are prefixed with # and only accessible inside the class. 
    
class BankAccount {
  #balance; // private
  constructor(initialBalance) {
    this.#balance = initialBalance;
  }
  deposit(amount) {
    this.#balance += amount;
  }
  getBalance() {
    return this.#balance;
  }
}

const account = new BankAccount(100);
account.deposit(50);
console.log(account.getBalance()); // 150
console.log(account.#balance); // SyntaxError
  • Why useful: Hides internal state from outside code, allowing controlled access and preventing misuse.

8. Error Handling

  • The try...catch...finally statement lets you handle runtime errors without stopping your program.

    • try: Place code that might throw an error.
    • catch: Handles the error if one occurs.
    • finally: Runs no matter what — useful for cleanup operations (closing connections, releasing resources).

    try {
  let result = riskyOperation();
  console.log("Operation successful:", result);
} catch (error) {
  console.error("Something went wrong:", error.message);
} finally {
  console.log("Cleanup actions executed");
}

  • Why useful: Prevents crashes and allows graceful recovery or fallback when something unexpected happens.

  • You can extend the built-in Error class to create your own error types. This makes error handling more specific and organized in large applications.

    class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}
              
function validateUser(user) {
  if (!user.name) {
    throw new ValidationError("Name is required");
  }
}
              
try {
  validateUser({});
} catch (err) {
  if (err instanceof ValidationError) {
    console.error("Validation failed:", err.message);
  } else {
    console.error("Unknown error:", err);
  }
}

  • Why useful: Helps categorize and handle errors differently depending on their type.

  • In async functions, errors are thrown just like in synchronous code — but must be caught using try...catch inside the async function or with .catch() on the returned Promise.

    async function fetchData() {
  if (Math.random() > 0.5) {
    throw new Error("Random failure");
  }
  return "Data fetched successfully";
}
              
(async () => {
  try {
    const data = await fetchData();
    console.log(data);
  } catch (error) {
    console.error("Error in async function:", error.message);
  }
})();

  • Why useful: Ensures asynchronous errors are handled properly, avoiding unhandled promise rejections and unexpected crashes.

9. Performance Optimization

  • Both are techniques to limit how often a function runs, improving performance when handling frequent events like scrolling or resizing.

  • Debouncing delays execution until a certain time has passed since the last call. Useful to avoid multiple quick triggers.

    function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
                  
 window.addEventListener('resize', debounce(() => {
  console.log('Resize event handled');
}, 300));

  • Throttling ensures the function runs at most once every set interval, ignoring calls during that period.

    function throttle(fn, limit) {
  let lastCall = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastCall >= limit) {
      lastCall = now;
      fn.apply(this, args);
    }
  };
}
                  
window.addEventListener('scroll', throttle(() => {
  console.log('Scroll event handled');
}, 300));

  • Why useful: Reduces unnecessary function executions, saving CPU and improving responsiveness.

  • Loads resources (images, scripts, modules) only when needed, usually when they enter the viewport or just before usage.

    <img src="placeholder.jpg" data-src="real-image.jpg" class="lazy">

  • JavaScript listens for scroll or uses Intersection Observer to swap in the real src when visible.

  • Why useful: Improves initial page load time and reduces bandwidth usage.

  • Minification: Compresses code by removing whitespace, comments, and shortening variable names without changing functionality.

  • Tree-shaking: Removes unused code during bundling (e.g., with Webpack or Rollup). Only code actually imported and used is included.

  • Why useful: Reduces file size, leading to faster downloads and better performance.

  • Memory leaks occur when your program retains references to objects no longer needed, preventing garbage collection.

  • Garbage collection is automatic in JavaScript; the engine frees memory by removing unreachable objects.

  • Common leak causes:

    • Global variables
    • Forgotten timers or event listeners
    • Closures holding large data unnecessarily

  • Why useful: Prevents increasing memory usage which can slow down or crash applications.

10. Testing & Debugging

  • console.log is the simplest way to output variable values, program flow, or errors to the console. It helps quickly identify where the problem is.

  • Browser DevTools provide powerful debugging tools including:

    • Inspecting HTML/CSS
    • Setting breakpoints to pause execution
    • Step-by-step code execution (step over, step into)
    • Watching variables and call stacks
    • Network requests monitoring
    console.log("Value of x:", x);
debugger;  // Pauses execution if DevTools are open

  • Why useful: Allows real-time inspection and control, making it easier to find bugs and understand code behavior.

  • Unit testing verifies small parts of your code (functions, modules) work correctly in isolation.

  • Popular JavaScript testing frameworks:

    • Jest: Zero-config, supports mocks and snapshots, widely used with React.
    • Mocha: Flexible test runner often combined with assertion libraries like Chai.
    test('adds 1 + 2 to equal 3', () => {
  expect(1 + 2).toBe(3);
});

  • Why useful: Ensures code correctness, catches bugs early, and makes refactoring safer.

  • A stack trace is a report of the active function calls at the point an error occurred.

  • It shows the call sequence (from the current function back to where execution started), file names, and line numbers.

  • Helps pinpoint the exact location and chain of calls that led to an error.

    TypeError: Cannot read property 'foo' of undefined
at bar (app.js:10:15)
at baz (app.js:20:5)
at main (app.js:30:3)

  • Why useful: Speeds up debugging by showing the path the program took before crashing.

11. Security

  • Cross-Site Scripting (XSS) attacks happen when malicious scripts are injected into trusted websites, potentially stealing data or hijacking user sessions.

  • To prevent XSS, always sanitize and escape user input before inserting it into the DOM.

  • Avoid inserting raw HTML from untrusted sources.

    const safeText = document.createTextNode(userInput);
element.appendChild(safeText);

  • Why useful: Protects users from malicious code execution in their browsers.

  • eval() executes a string as JavaScript code, which can lead to serious security vulnerabilities if the string contains untrusted data.

  • Avoid eval() completely or replace it with safer alternatives like JSON.parse() or functions.

  • Why useful: Prevents execution of arbitrary, potentially harmful code.

  • Using innerHTML to insert HTML content can open doors to XSS if the content isn’t sanitized.

  • Prefer textContent for inserting text safely, or sanitize HTML strings before use.

  • Use libraries like DOMPurify if you must insert HTML from external sources.

    element.textContent = userInput; // safer than innerHTML

  • Why useful: Prevents injecting unsafe HTML or scripts into the page.

  • Always use HTTPS to encrypt data in transit.
  • Validate and sanitize all input on the server side, not just client side.
  • Use authentication tokens (like JWT) securely and avoid exposing secrets in client code.
  • Implement rate limiting and CORS properly to prevent abuse.

  • Why useful: Protects sensitive data and prevents unauthorized access.
